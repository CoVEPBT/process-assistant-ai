Polish + safety net pass: anchored inline flags, prompt invariants,
              post-process drift detection, title cap, preflight

Bundled 1 Jun 2026 pilot iteration into one commit. Covers everything
on top of 1bc2afe (Tracker #25 + #28).

1. Inline view: anchor more flag types to the right place
========================================================
- Handover flags render between the two activities they sit between,
  as an amber strip with click-to-expand suggestion body, instead of
  the process-wide banner.
- NOTE flags (note-question, note-empty) now anchor to the specific
  NOTE element. Note flag location includes "NOTE N", resolveJumpTarget
  matches it before the activity pattern and returns doc-note-{X}-{Y}-{ni},
  pillLabel renders as "NOTE 5.0.2" instead of "Activity 5.0".
- Empty-role placeholder gets the same id="doc-activity-X-Y-role" as
  populated roles, so act-norole-N warnings anchor on the
  "[Click to add accountable role]" placeholder rather than falling to
  the process-wide banner.

2. Inline flag visual redesign
==============================
- Dropped the full background-colour highlight on flagged elements.
  Was clashing with active-task highlighting.
- Dropped the "click for detail" pill that hung off the end.
- New treatment: circular severity-coloured icon (!/error red,
  triangle/warning amber, i/info blue) before the flagged text, plus
  dashed underline at currentColor. Subtle, less competing with the
  task content.
- Per-severity backgrounds set explicitly because currentColor on a
  ::before resolves to the same color set in the block (white) and
  rendered the icon invisible.

3. Inline flag click reliability
================================
- Per-element click handlers were closure-captured idx/issue and
  went stale after Save changes re-ran the rules engine. Replaced
  with one delegated click + keydown listener on docPreview that
  looks up the current issue from currentIssues[dataset.issueIdx]
  at click-time. New tasks added via "+ Add task" -> save -> click
  now works.
- Callouts get contenteditable="false" so they don't get caught up
  in the doc's editable parent.

4. Inline-mode UX cleanup
=========================
- "Tip: click a card to jump..." hides in In-doc mode.
- Severity-upgrade-only logic in applyInlineFlags: when multiple
  flags target the same element, keep only the HIGHEST severity
  class (error > warning > info). CSS source order was silently
  downgrading warning-amber highlights to info-blue.

5. Quality flag card layout + content
=====================================
- "click for detail" pill (long-task helper) - now padding 3px 10px 4px,
  vertical-align middle, line-height 1.6, true pill border-radius
  999px; hover keeps white bg + thicker border, no colour collision.
- buildLongTaskSplitSuggestion helper for >30-word tasks: pulls the
  first sentence as the action, scans the rest for a named guide/QRG/
  SOP/manual/handbook/procedure/policy, emits a concrete "Consider
  splitting: keep <action>, move detail to a NOTE referencing <guide>"
  suggestion.
- Card body de-duplication via stripRedundantPrefix(): "Activity 5.0
  title is 8 words" -> "Title is 8 words"; "Task 5.0a is 38 words" ->
  "Is 38 words". Pill and message no longer say the same thing twice.
- row1 order swap: pill on left (where), severity badge on right (how
  serious), dismiss at end.
- Quality Flags sidebar header alignment: removed orphan
  .flags-view-toggle / .flags-view-btn CSS, dropped margin-top:6px
  offsets, added white-space:nowrap so "In doc" doesn't break onto two
  lines, added flex-wrap to .doc-sidebar-header.
- extractText now strips .user-item-remove / .doc-add-actions /
  .doc-add-btn before reading innerText. Stops the dismiss-button "x"
  character bleeding into saved task text and tripping the verb-first
  check ('starts with "x", which isn't on our verb list').

6. Pre-analyse validation
=========================
- Full parse + validation runs BEFORE the consent modal opens (was
  only catching the truly-empty case). Empty paste, missing title,
  no detectable activities in an uploaded file - all now surface their
  toast before the modal. Parsed model is cached on
  window._preflightModel so _doAnalyse reuses it without re-parsing.

7. Post-analyse: hide scaffolding until analysis completes
==========================================================
- Removed the "I've raised X quality flags... Choose how you want to
  handle them" headline entirely on the non-empty case.
- No more scaffolding flash before auto-apply. Immediately after
  renderResults() and before the API call kicks off, postAnalyseDetail
  + choiceRow + flagHeadline get hidden and autoApplyLoading shown.
  Trigger the API on the next paint frame (50ms instead of 400ms).
  Net effect: when the consent modal closes, the user sees only the
  loading spinner.

8. Parser: structure detection
==============================
- Flat top-level numbering becomes separate activities. Pre-process
  splits inline "N. text. M. text" runs onto separate lines, then
  allFlatNumbered detection (every action line is top-level "N. text"
  with no decimals/letters/bullets) treats each as a new activity.
- Tightened with two nesting signals (indented numbered lines OR
  number repeats) so legitimate nested numbering doesn't get flipped.
- Separator-style hierarchy signal: when activities use one terminator
  ("1.") and tasks use another ("1)"), a new line matching the
  activity terminator after tasks in the OTHER terminator is treated
  as a new activity. Catches "1. Prepare meal / 1) Get ingredients /
  2) Cook / 2. Eat meal" patterns.

9. Auto-apply prompt: invariants + activity heading rules
=========================================================
- Activity heading length cap raised 7 -> 10 words across the rules
  engine, all UI copy, Word/PDF generator strings, and the prompt
  block. NOTE: was 2-7, now 2-10.
- "Hard cap: 8 words" contradiction removed. Now reads strict 2-10,
  "NO EXCEPTIONS. 11 words = FAIL. 12 words = FAIL." plus the
  framing "Do not skip trimming because you respect the source -
  the source IS the thing you are correcting."
- 5th self-check step: emitting a heading >10 words is task failure.
- ACTIVITY COUNT INVARIANT added (mirrors NOTE count): forbids
  merging two source activities into one. Prepare meal / Eat meal
  case as the worked example.
- TASK COUNT INVARIANT added: "If the source has 4 tasks the output
  has at least 4 tasks. If you cannot fit a task under the activity
  title, change the activity title - do NOT drop the task." Spaghetti
  regression as the worked example.
- NOTE COUNT INVARIANT strengthened on the zero case: "If the source
  has ZERO NOTEs under an activity, the output also has ZERO NOTEs..."
- SPECIFICS BELONG IN THE BODY rule: when a heading is loaded with
  legal references / section numbers / Act names, those go into tasks
  or NOTEs - not the heading. AHPRA Information request case as the
  worked example.

10. Post-process safety net (#223)
==================================
Deterministic, prompt-independent check that runs after auto-apply
parses the AI response. Compares source vs output and surfaces a
warning toast (10s) if:
  - activity count dropped
  - total task count dropped
  - total NOTE count grew (invented NOTEs)
  - any activity title still over 10 words

Console.warn diagnostic logs the actual counts + over-cap titles for
engineering visibility. Toast text:
  "Heads up: auto-apply produced [N tasks missing, M new NOTEs added,
   K titles over 10 words] compared with your original. Spot-check
   the doc and use 'Restore original' if needed."

User keeps control - the safety net warns but doesn't auto-revert.
This is the deterministic backstop for the spaghetti / prepare-meal /
AHPRA Information request regressions that we kept trying to fix
prompt-side.

11. Rules engine: severity rebalance + plumbing
===============================================
- act-length (activity title length) upgraded from info to warning so
  all activity-title flags read as the same family of issue.
- Long-task info suggestion (>30 words) generates an actionable prompt
  via buildLongTaskSplitSuggestion.
- no-trigger / no-outputs / no-inputs flags REMOVED. Empty Trigger /
  Inputs / Outputs fields now show "To be assigned in Nintex Process
  Manager" placeholder, mirroring the owner/expert pattern. Field
  stays contenteditable. extractModelFromDocPreview treats the
  placeholder text as empty on save.
