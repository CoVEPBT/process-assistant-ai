Auto-apply: restore activity numbers + linked processes

Two post-process restores added to the auto-apply pipeline.

#183 — Activity numbers (6.1 / 6.2 / 6.3)
The PRESERVE ACTIVITY STRUCTURE prompt rule from a3940de tells
the AI to keep sub-decimal numbers verbatim ("6.1" not "7.0"),
but the model still flattens them on some runs. Prompt alone
clearly isn't enough.

New restore pass mirrors the existing role-restore pattern:
walk new activities in order, overwrite each one's number
with the original's number by position. Capped at
min(newCount, origCount) so a legitimate multi-role split
that adds activities doesn't trip the loop. Now 6.1 / 6.2 /
6.3 survive the rewrite by force.

#184 — Linked processes (act.links / "PROCESS X" lines)
The parser captures "PROCESS X" lines under each activity
into act.links. The three-block AI output format doesn't
include them, so any linked-process reference disappears
after auto-apply. The export then shows nothing under the
activity it should belong to.

New restore pass: match new activities to original activities
by activity NUMBER first (now reliable because of #183
above), fall back to index. If the new activity has no links
but the original did, copy the original list over. Doesn't
ADD links the source didn't have - only carries forward what
was already there.

Both passes log to console so we can spot how often the AI is
still messing this up.
