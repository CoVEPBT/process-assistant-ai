Morning Schedule-new-class test: three fixes

Natalie's morning test of the Schedule-new-class process
(pasted Nintex Procedure Text into the tool, ran auto-apply)
surfaced three real issues:

1. AI changed activity 1's role list.
2. Tasks not rewritten to verb-first ("If the request is to
   cancel a class, action in SAMS" stayed as-is across all 5
   tasks in activity 2.0).
3. Multi-role quality flag mis-firing: false-positive on
   single roles that contain an "&" inside the title (e.g.
   "CoVE - Planning & Admissions" treated as two roles), and
   missing real comma-separated multi-role lists (e.g.
   "CoVE - Program Coordinator, CoVE - Program Manager,
   CoVE - Student Lifecycle" treated as one).

Three fixes:

A. SMARTER splitRoles (issue #3)
   The old function naively split on ";", " & ", " and ",
   " + ", " / ", " \ ". That mis-handled roles with those
   tokens inside the role title.

   New logic: hard-split on ";" only. For ",", " & ",
   " and ", etc. — only split if the part AFTER the separator
   looks like the start of a new role expression (starts with
   an org prefix like "CoVE -", "RMIT -", "DP&S -" etc, or a
   capitalised noun + dash + role keyword).

   Unit-tested 11 real RMIT cases including:
     "CoVE - Planning & Admissions"        -> 1 role
     "CoVE - Coordinator, Student Lifecycle" -> 1 role
     "Customer and Service Officer"        -> 1 role
     "DP&S - Quality Officer"              -> 1 role
     "CoVE - X; CoVE - Y; CoVE - Z"        -> 3 roles
     "CoVE - X, CoVE - Y"                  -> 2 roles
     "CoVE - X and CoVE - Y"               -> 2 roles
     "CoVE - X & CoVE - Y"                 -> 2 roles
   11/11 pass.

B. VERB-FIRST FOR EVERY TASK (issue #2)
   New non-negotiable rule in BOTH the frontend auto-apply
   prompt and the server SYSTEM_PROMPT. Includes worked
   examples for the three patterns Natalie hit:
     - Conditional opener: "If the request is to cancel a
       class, action in SAMS" -> "Cancel the class in SAMS
       when the request is a cancellation" (or split to task
       + NOTE "When does this task apply?").
     - Descriptive opener: "Review request ensuring all
       fields are populated" -> "Confirm all fields relevant
       to the request are populated".
     - Noun opener: "Email sent to the employer" -> "Email
       the employer".
   Lists invalid first words (If, When, Where, The, All, A,
   An, any noun, any -ing participle, present-tense is/are)
   and a mandatory self-check that walks every task before
   emitting the Process Map block.

C. ROLE RESTORE by activity number (issue #1)
   The role-restore post-process previously matched new tasks
   back to original tasks by ACTIVITY INDEX. Same problem we
   hit for attachments and linked processes: any AI activity
   reorder breaks the mapping. Now matches by activity NUMBER
   first (reliable thanks to the activity-number restore
   pass), falls back to index for legacy cases. Same defence-
   in-depth pattern as 70271ee.
