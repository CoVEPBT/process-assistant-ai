Stop AI emitting both original + polished version of same NOTE

Item #5 from the post-a3940de export test: the AI polished a
source single-directive NOTE under activity 2.0 ("Signature
can only be entered in SAMS after the delivery schedule has
been confirmed by CoVE Program Manager") into a Q+A NOTE
("When can the signature be entered in SAMS? / Signature can
only be entered..."), but ALSO kept the original directive
NOTE as a second title-only NOTE alongside. Source had 1
NOTE in that activity; output had 2.

Existing PRESERVE NOTES count-check should have caught this
in the AI's self-check but didn't. Two changes:

1. PROMPT - strengthened PRESERVE NOTES rule:
   - Explicit "ONE-TO-ONE MAPPING" subsection: each source
     NOTE maps to EXACTLY one output NOTE. The polished
     version REPLACES the original; it does NOT sit alongside.
     If two output NOTEs would have overlapping body text,
     delete the duplicate before emitting.
   - "DIRECTIVE NOTES ARE OK AS-IS" subsection: A NOTE does
     not have to be Q+A. Source NOTEs that are single
     directive statements can stay as a single statement on
     the title line - do not invent a question to wrap them.
     This stops the AI feeling pressure to create Q+A
     versions that then duplicate the source content.
   - Sharpened the count self-check wording with a concrete
     example: "Source has 3 NOTEs under activity 4? Output
     has exactly 3, NOT 4, NOT 2."

2. POST-PROCESS - within-activity duplicate-NOTE dedupe.
   After NOTE restoration, scan each activity's NOTEs and
   detect duplicates: two NOTEs where one's title is
   substantially the same content as the other's body, or
   where bodies overlap. The substantial-overlap test
   requires the shorter string to be 25+ chars and fully
   contained in the longer one - that catches the classic
   "directive kept alongside its Q+A polish" pattern without
   false-positiving on incidental short word collisions.
   Drops the duplicate, keeps the more informative NOTE.
