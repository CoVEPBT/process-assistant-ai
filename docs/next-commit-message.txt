Tracker #52: Re-run auto-apply after user edits

Lets users re-run auto-apply on top of their edits to the doc
preview, without losing the ability to "Restore original"
back to the very first parsed state.

What changes for the user
- After the first auto-apply completes, the top-tab "Re-run
  auto-apply" button stays visible (was hidden permanently
  before). They can edit the doc preview - add tasks/notes,
  fix wording, etc. - and click Re-run auto-apply to apply
  the AI polishing pass again on the current content.
- "Restore original" ALWAYS goes back to the very first
  parsed state. Never to a previous auto-apply baseline.
  The safety net is anchored to ground truth, not to the
  last AI output.
