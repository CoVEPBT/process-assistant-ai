Fix: skip auto-structure offer for Nintex procedure-text paste

Natalie pasted content in Nintex's "Export Procedure Text" format
(activity headings like "Title [Role A;Role B]" + 6-space indented
tasks). The looksLikeRoughNotes() detector only checked for numbered
headings ("1.0 Title", "Activity 1"), so it incorrectly classified
the already-structured procedure-text format as "rough notes" and
offered to AI-structure it. The AI then tried to restructure already-
structured input and produced garbled output - activity titles got
an "a " prefix and the tasks disappeared.

Fix: looksLikeRoughNotes() now calls looksLikeProcedureTextExport()
first; if true, it returns false (already structured, no need to
re-structure). The Analyse flow then routes the paste straight to
parseStructuredText() which auto-detects the procedure-text format
and dispatches to parseProcedureTextExport() for clean parsing.

Reproduction: paste content of the form
    Title 1 [Role A;Role B]
          Task one
          Task two
    Title 2 [Role C]
          Task three
- before this fix, auto-structure modal opens and re-mangles the
  text on click-through
- after this fix, the modal is suppressed; Analyse goes straight
  through and parseProcedureTextExport handles it correctly.
