CRITICAL FIX FIRST
==================

Repaired a JavaScript syntax error introduced in yesterday's
commit 0c5e778 (the verb-first prompt rule).

What was wrong
--------------
The VERB-FIRST FOR EVERY TASK block in the auto-apply prompt
had multiple bugs:

  - Several string literals were missing the trailing "+"
    operator that concatenates them into the prompt.
  - One line had an unterminated string - the closing quote
    was dropped mid-rule.

JavaScript treated this as a SyntaxError at parse time, which
means the WHOLE inline <script> block containing it stopped
executing on page load. Everything defined further down in that
block - which includes:

  - startAutoApply() and the auto-apply pipeline
  - role / activity-number / linked-process restore passes
  - duplicate-NOTE dedupe
  - trigger / trailing-requirement drift detection
  - the entire post-process safety net we built up over the
    last three days

- was almost certainly not loading on the live site.

How I caught it
---------------
Ran `node --check` against every inline <script> block in
public/index.html. The verb-first block failed parse. Fixed
the missing "+" operators and the unterminated string. Re-ran
the check: all 12 blocks now parse cleanly. server/index.js
also clean.

Then verified the auto-apply prompt builds at runtime
(25,137 chars) and contains every rule we expect:
PRESERVE ACTIVITY STRUCTURE, ONE-TO-ONE MAPPING (NOTEs),
CONDITIONAL / TRIGGER CLAUSES, TRAILING REQUIREMENT SENTENCES,
PRESERVE URLS, VERB-FIRST FOR EVERY TASK, and the worked
examples (e.g. "Cancel the class in SAMS when...").

Re-ran the splitRoles unit tests too: 11/11 pass.

If the deployed version has been throwing a console error since
yesterday's push, this fixes it. Worth opening DevTools on the
live site before deploying this to confirm.


Tracker #49 - Word export missing the checklist link
====================================================

The "Open the CoVE Process Review Checklist" hyperlink at the
end of the Word export wasn't rendering. Two issues:

  1. The TextRun inside ExternalHyperlink referenced
     style: 'Hyperlink' - but no character style with that id
     was defined in the Document's styles config. The reference
     resolved to nothing and the text disappeared.

  2. The TextRun's underline was `underline: {}` - probably
     under-specified for docx@8.5.0 and rendering nothing.

Fix:

  - Added a `characterStyles` array to the Document styles
    config, defining a "Hyperlink" style (blue + underlined).
  - Tightened the TextRun underline to
    `{ type: 'single', color: '1A56DB' }`.

Belt and braces - the hyperlink now renders even if either the
style lookup or the explicit underline alone failed.
