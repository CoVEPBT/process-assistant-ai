PAT export test feedback: items #1 #2 #3 #4

Four issues Natalie hit running the Training Plans PDF through
auto-apply then exporting to PDF.

#1  AI invented an activity 5.0 "Follow Up with Employer" out
    of NOTE content under source activity 4.0 (the
    "no response" / "still no response" follow-up emails). The
    invented activity also duplicated one of the source NOTEs.

#4  AI flattened source activities 6.1 / 6.2 / 6.3 (parallel
    activities under 6.0) into whole-number 7.0 / 8.0 / 9.0,
    losing the parallel-activity semantics. The x.y numbering
    in Nintex specifically means "these are alternatives, not
    sequential steps".

#3  SharePoint URL attached to source activity 6.1 ("SAMS -
    Enter Employer Sign-off Is Not Required For A Course")
    disappeared in the export. Root cause was the off-by-one
    from #1: the attachment-restore pass matched new tasks
    back to original tasks by ACTIVITY INDEX, and the invented
    activity shifted everything by one.

#2  Yellow-bordered NOTE blocks in the PDF and Word exports
    had too little top padding - the first line of the NOTE
    title sat tight against the border.

Fixes:

1. New PRESERVE ACTIVITY STRUCTURE rule in BOTH the frontend
   auto-apply prompt and the server SYSTEM_PROMPT.
     - Never invent new activities.
     - Never promote NOTE content into a separate activity.
     - Preserve activity numbers verbatim, INCLUDING sub-
       decimals like 6.1 / 6.2 / 6.3.
     - Never duplicate a NOTE across multiple activities.
     - Mandatory self-check: count source activities, count
       output activities, verify each output number matches a
       source number exactly.

2. Strengthened PRESERVE URLS rule in the frontend prompt to
   tell the AI explicitly about the "Link: https://..." row
   the source view renders below tasks with attachments -
   that row IS the attachment and must be kept under the same
   task in the output.

3. Defence-in-depth in the attachment-restore pass: also
   match original activities by activity NUMBER, not just by
