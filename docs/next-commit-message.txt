AI guardrails: fix items #32 (empty task before Link) + #33 (dropped qualifiers)

Two regressions caught in yesterday's BARB/FLOSS + Class Scheduling tests.

ITEM #32 - Auto-apply leaves empty task before a Link row
  Symptom: source task like "[Web Link] Trades & Dental Weekly TT" with
  an attached URL was being rewritten so that the task ended up as a
  bare letter ("c.") followed by just a "Link:" row below. URL post-
  process restoration re-attached the URL but the task text was empty.

  Fix - three layers:
    a) Auto-apply prompt: NEVER LEAVE A TASK EMPTY rule. Tells the AI
       that "[Web Link] Trades & Dental Weekly TT" becomes a task with
       the URL attached AND descriptive text "Trades & Dental Weekly TT"
       (drop the [Web Link] marker, keep the words). Never emit a bare
       task letter with no descriptive text.
    b) Server SYSTEM_PROMPT mirrors the rule for chat-driven rewrites.
    c) Post-process restoration extended: when the new task text is
       empty / whitespace but the original had text, restore the source
       text (with the [Web Link] marker stripped). Also defensively
       cleans the [Web Link] marker from any new task text where the
       AI leaked it through.

ITEM #33 - Auto-apply dropping in-task qualifiers / specifics
  Symptom: despite the PROMOTE DROPPED CONTEXT TO NOTES rule, the AI
  was still removing specific details when tightening sentences:
    - "(continuing and new)" parenthetical qualifier
    - "3.0 & 4.0" step number references
    - "A&T schedule in SAMS" -> generic "set attendance dates"
    - "as well as any future changes" additional action

  Fix - two layers:
    a) Prompt: new EXAMPLES YOU MUST GET RIGHT block showing SOURCE /
       WRONG / RIGHT for each of the four patterns above. Mandatory
       self-check now enumerates four explicit checks in order:
         1. Did I drop a parenthetical "(...)" qualifier?
         2. Did I drop a specific number reference (step 3.0, section 4)?
         3. Did I drop or generalise a system / table / view / tool name?
         4. Did I drop an additional action joined by "and" / "as well as"?
       Server SYSTEM_PROMPT carries the same enumeration for chat-driven
       rewrites.
    b) Post-process drift detection: after parsing the AI's reply, the
       client scans every task for dropped parent
