17. Process title mangled
    Source: file 2 #8
    Status: [DONE]
    Implementation: rewrote the Nintex PDF title detector. The old
    regex required an uppercase acronym after "CoVE - " (e.g.
    "CoVE - SL -") so it skipped the real title "CoVE - Student
    Lifecycle - ..." and matched the swimlane role label instead.
    New approach anchors on the status marker ("[ Awaiting
    Publish ]" / "[ Approved ]" etc.) or version "vX.Y", iterates
    every candidate in the first 50 lines, walks back up to 5
    lines with hyphenation-aware word-wrap joining (so "regis-"
    + "tration" -> "registration"), and prefers candidates that
    start with "CoVE -" or "RMIT -" - which correctly skips the
    isolated status-badge fragment produced by the page-1 column
    split. Verified end-to-end against the AHPRA PDF and the
    TeachVE PDF.

18. Roles bleeding into activity titles
    Source: file 2 #11
    Status: [DONE]
    Implementation: extended the role-line detector in
    parseProcedureBlock. The old roleLineRe only matched lines
    starting with "CoVE"/"RMIT" or containing en/em dashes, so
    plain-hyphen role labels like "STEM - SHBS - Accreditation
    Manager" or "USARG Government Reporting - Manager" fell
    through to the title-continuation branch and concatenated
    onto the activity title. New isRoleLine() also accepts
    "ALL-CAPS prefix + plain dash + role-keyword (Manager,
    Officer, Coordinator, Analyst, etc.)". Verified end-to-end -
    all 9 activities in the AHPRA PDF now show the correct title
    and role with no bleed.

23. Dismiss a warning
    Source: file 1 #7
    Status: [DONE]
    Implementation: each quality-flag card gets a "x" dismiss
    button (top-right of the card). Click it and the flag is
    hidden from the sidebar, drops out of the "N to review"
    count, the summary chip, the Export error gate, AND the
    ISSUES FLAGGED block sent to the AI (so the AI doesn't
    keep advising on flags the user has set aside).

    Per-session only - no localStorage. Resets / Analyse another /
    new Analyse clear the dismissed set. A "N dismissed - Restore
    all" footer appears in the sidebar whenever at least one flag
    is dismissed, so the user can bring everything back in one
    click. Each individual dismissal also shows a toast with an
    "Undo" button (6s window) for accidental clicks.

    Flag identity = id + location + message, so dismissals
    survive Save changes re-runs as long as the flag still
    matches. If the user fixes the underlying issue, the rule
    stops firing and the dismissal naturally falls away.

31. Auto-apply was dropping NOTE blocks + activity titles too long
        + AI was rewriting/deleting roles
        + AI was deleting useful context when shortening tasks
    Source: Natalie 27 May 2026 (regressions caught post-#23 testing,
            FRANC document diff screenshot + RPO/Staff Name example)
    Status: [DONE]
    Implementation: four AI behaviour fixes in the auto-apply path:
      a) Auto-apply prompt now has an explicit PRESERVE NOTES
         block ("NOTEs are load-bearing, count source vs output,
         redo if mismatched") + post-process restoration that
         puts back any NOTE the AI still drops. Fuzzy match by
         first 40 normalised chars of the title so the AI's
         polish wording doesn't trigger a false "missing" hit.
      b) Auto-apply prompt now has an explicit ACTIVITY HEADING
         LENGTH block (2-7 words target, hard cap 8 words, with
         self-check requirement). Server-side SYSTEM_PROMPT
         also strengthened with the same cap so chat-only
         rewrites don't drift past 8 words either.
      c) Auto-apply prompt now has an explicit PRESERVE ROLES
         block (verbatim copy, character-for-character, including
         punctuation/abbreviations/ampersands/capitalisation).
         Server SYSTEM_PROMPT also has a "preserve role list
         verbatim" rule. Post-process step copies the original
         role string back per activity if the AI altered it -
         covers cases like "CoVE - Delivery Planning & Support,
         CoVE - Quality & Compliance" being rewritten to
         "CoVE - Delivery Planning Support".
      d) Auto-apply prompt now has a PROMOTE DROPPED CONTEXT
         TO NOTES block - when shortening a task to hit the
         18-word cap, the AI must not silently delete specific
         nouns (column names, section names, criteria, etc.).
         Instead it should keep the task tight and create a new
         NOTE that captures the dropped detail. Worked example
         baked into the prompt from Natalie:
           SOURCE: "Allocate required staff to RPO grade rosters
                   referring to the data noted Staff Name
                   columns in the FRANC document"
           WRONG : "Allocate staff to grade rosters from the
                   FRANC" (lookup detail lost)
           RIGHT : Task = "Allocate required staff to RPO grade
                   rosters using the FRANC document"
                   NOTE = "Where do I find the staff data in
                   the FRANC? / Look in the Staff Name column."
         Server SYSTEM_PROMPT has the same rule for chat-driven
         rewrites.

30. Admin "review pack" export - doc + quality flags together
    Source: Natalie 27 May 2026
    Status: [DONE]
    Implementation: new "Review pack (.docx)" button in the Export
    modal, under a new "Admin actions" group with a small navy
    "Admin" badge. Generates a single .docx containing:
      - the polished process document (title, metadata, objective,
        background, activities with roles, tasks with letters
        restarted per activity, NOTE blocks, FORM/PROCESS links,
        preserved task URLs)
      - each ACTIVE quality flag (dismissed flags excluded per
        item 23 design) rendered as a Word REVIEW COMMENT
        anchored to the relevant activity heading, task, or
        process title
      - "For: process owner / expert" intro panel with reviewer
        + date + flag count
      - Owner sign-off block at the end for name / signature /
        date
    The flag-to-paragraph mapping uses the issue location string
    (Task 2.0a -> task paragraph; Activity 2.0 -> heading;
    title-* -> title paragraph). Process-wide flags fall back to
    the first activity heading so nothing is silently dropped.
    Built client-side using docx-js 8.5.0's native CommentRangeStart/
    CommentRangeEnd/CommentReference + comments.children API.
    Verified end-to-end in node: 3 sample flags -> 3 comments in
    comments.xml + 9 markers (start/end/reference x 3) in
    document.xml. Real .docx open + show comments correctly in
    Word desktop.

    Naming: "Review pack" chosen as user-facing label. Distinct
    from item 29 (flags-only export) which stays separate as the
    lighter sharing format.
